## Metodologia de Análise – Reconnaissance
Todas as análises deste projeto são feitas sob perspectiva de um analista SOC/Blue Team. <br/>
É um projeto de nível teórico utilizando o framework da MITRE, ATT&CK.

*Esse projeto foi feito com intensão de mapear táticas de reconhecimento e é deliberadamente limitado, servindo melhor apenas como aprendizado e portfólio*

## Por que Reconnaissance foi a tática escolhida?
A fase de reconhecimento é o instante que o atacante quer apenas coletar informações dos sistemas. Nessa fase, a intenção do atacante é se manter invisível enquanto faz o reconhecimento dos padrões do sistema envolvido, logo, reconhecer qualquer ação suspeita de mínima intensidade como sendo de um usuário malicioso, seja por varreduras ou outras análises, é a chave para impedir um ataque que gere ainda mais prejuizos à empresa. Quanto mais cedo neutralizar a ameaça, sob maior segurança o sistema é mantido.

## Critérios de Análise
- Intenção do atacante: Se quer descobrir portas acessíveis, nomes de usuários, entre outras informações;
- Nível de ruído: Se a ação realizada é realmente maliciosa ou se apenas parece suspeita mas é legítima;
- Visibilidade para o SOC: Se uma ação é de difícil detecção ou muito clara;
- Impacto Defensivo: Se não perceber a ação suspeita, quais seriam os impactos defensivos na organização.
